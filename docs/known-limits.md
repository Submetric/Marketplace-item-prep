# Known Limits

This page describes the current public scope for `Lite` and `Pro`.

## Current Product Boundary

Marketplace Item Prep currently focuses on creator-side thumbnail prep inside Roblox Studio.

The public `Lite` and `Pro` docs cover:

- scene setup
- framing
- backdrop and lighting control
- reviewed shot selection in `Pro`
- reusable named scene setups in `Pro`

## What The Plugin Does Not Claim

Marketplace Item Prep does not claim:

- guaranteed Marketplace approval
- guaranteed moderation success
- automatic publishing on your behalf
- guaranteed final thumbnail parity in every case

## Tool Boundary

Marketplace Item Prep is not a replacement for:

- Roblox 3D Importer
- Accessory Fitting Tool
- full validation workflows
- Roblox Marketplace publishing flow

It sits after fitting and validation, then helps with thumbnail preparation and scene review.

## Lite And Pro Scope Limits

The current public docs for `Lite` and `Pro` do not cover:

- repeated carry-forward workflows
- cross-item scene consistency systems
- official Marketplace thumbnail handoff
- upload-unit classification
- family-aware Marketplace workflow support
- submit-ready checks

Those areas are intentionally outside the current public Lite/Pro documentation scope.

## Preview And Stage Limits

`Stage Shot` creates plugin-owned preview content for composition work.

That means:

- preview work is meant to stay separate from silent source mutation
- the stage can be refreshed, returned to, or cleaned up through plugin actions
- the stage is a preview workflow, not a publishing or export workflow

## Selection Limits

For the current public workflow, the plugin is documented conservatively around selected:

- `Accessory`
- `Model`
- `MeshPart`

If the current selection is not usable, the plugin should guide you back to a valid target instead of creating a broken stage flow.

## Pro Positioning Limit

`Pro` is the single-item quality tier.

It improves the current shot through better controls, more polish, and lower framing risk. It is not the public workflow tier for repeated multi-item operations.

## If Something Looks Wrong

Try this first:

1. confirm the selection is valid
2. press `Frame`
3. switch between `World Shot` and `Stage Shot`
4. use `Reset` if the camera feels over-adjusted
5. clean the stage and rebuild the preview session

If the problem continues, open the plugin in Roblox Studio and use `Help > Contact Support`.
